Python Basics

Syntax, variables, data types and operators

Why This Matters

Python basics are the foundation of all programming. Just like learning the alphabet before writing, understanding variables and data types is essential for data science. You'll use these concepts every time you work with data in Python.

Code Example

variables.py 
# Variable assignment examples
student_name = "Chinedu Okoro"  # String
student_age = 24            # Integer
cgpa = 3.8                # Float
is_graduated = False        # Boolean

# Arithmetic operations
years_to_graduate = 4 - (student_age - 20)
total_score = cgpa * 25  # Convert CGPA to percentage

# Print formatted information
print(f"Student: {student_name}")
print(f"Years to graduation: {years_to_graduate}")
print(f"Total Score: {total_score}%")

Code Explanation

  • student_name = "Chinedu Okoro" - Creates a string variable with student's name
  • student_age = 24 - Creates an integer variable for age
  • cgpa = 3.8 - Creates a float (decimal) variable for CGPA
  • is_graduated = False - Boolean variable (True/False)
  • years_to_graduate = 4 - (student_age - 20) - Calculates years until graduation
  • total_score = cgpa * 25 - Converts CGPA to percentage scale
  • print(f"Student: {student_name}") - Prints formatted output with variable value

Real-World Analogy

Think of variables as labeled storage boxes in a Nigerian market. The variable name is the label on the box (e.g., "Yam Tubers"), and the value is what's inside the box (e.g., 50 tubers). Data types define what kind of items can be stored - numbers for quantities, text for descriptions, and true/false for availability status.

Control Flow

Conditionals, loops and logic

Why This Matters

Control flow allows your programs to make decisions and repeat actions. In data science, you'll use conditionals to filter datasets, loops to process thousands of records, and logical operators to combine complex conditions.

Code Example

control_flow.py 
# Nigerian university grading system
scores = [72, 85, 63, 90, 55]
passed = []

# Loop through scores with conditional logic
for score in scores:
    if score >= 70:
        passed.append(score)
        print(f"Score {score}: Passed")
    elif score >= 50:
        print(f"Score {score}: Passed with condition")
    else:
        print(f"Score {score}: Failed")

# While loop example
count = 0
while count < 5:
    print(f"Processing student {count+1}")
    count += 1

Code Explanation

  • scores = [72, 85, 63, 90, 55] - Creates a list of student scores
  • passed = [] - Creates an empty list to store passing scores
  • for score in scores: - Loops through each score in the list
  • if score >= 70: - Checks if score is 70 or higher
  • passed.append(score) - Adds passing scores to new list
  • elif score >= 50: - Checks for conditional pass (50-69)
  • else: - Handles failing scores (below 50)
  • while count < 5: - While loop that runs until condition is false

Real-World Analogy

Control flow is like a traffic officer directing vehicles at a busy Lagos intersection. Conditionals (if/else) are the officer deciding which cars can go based on conditions. Loops are like the continuous flow of traffic during rush hour, processing each vehicle one after another until the road is clear.

Functions

Creating reusable code blocks

Why This Matters

Functions allow you to write reusable code blocks, making your programs more organized and efficient. In data science, you'll create functions for data cleaning, transformation, and analysis that you can reuse across multiple projects.

Code Example

functions.py 
# Function to calculate Nigeria's GDP growth
def calculate_growth(current_gdp, previous_gdp):
    """Calculate percentage growth between two GDP values"""
    growth = ((current_gdp - previous_gdp) / previous_gdp) * 100
    return round(growth, 2)

# Function to analyze economic sector performance
def sector_performance(sectors):
    """Analyze and print sector performance"""
    print("\nSector Performance Analysis:")
    for sector, growth in sectors.items():
        if growth > 5:
            print(f"- {sector}: ▲ Strong growth ({growth}%)")
        elif growth > 0:
            print(f"- {sector}: ↗ Moderate growth ({growth}%)")
        else:
            print(f"- {sector}: ▼ Decline ({growth}%)")

# Using the functions
gdp_2022 = 440  # Billion USD
gdp_2023 = 477  # Billion USD

growth_rate = calculate_growth(gdp_2023, gdp_2022)
print(f"Nigeria GDP Growth: {growth_rate}%")

sectors = {
    "Agriculture": 3.2,
    "Oil & Gas": -1.5,
    "Technology": 8.7,
    "Manufacturing": 2.1
}
sector_performance(sectors)

calculate_growth: A function that calculates Nigeria's GDP growth rate by comparing the current GDP with the previous year's GDP.

sector_performance: This function analyzes how well each sector of the economy performed. It prints a summary with icons: ▲ for strong growth, ↗ for moderate growth, ▼ for decline.

Example: GDP grew from $440B to $477B in 2023, so the function will return the growth rate.

gdp_2022 = 440
gdp_2023 = 477
growth_rate = calculate_growth(gdp_2023, gdp_2022) ➝ 8.41%

Sector Output: Technology showed strong growth, Oil & Gas declined, while Agriculture and Manufacturing had moderate growth.

Data Structures

Lists, dictionaries, tuples and sets

Lists

Ordered, mutable collections

Lists are like expandable file folders - you can add, remove, and modify items at any position.

states = ["Lagos", "Kano", "Abuja", "Rivers"]
states.append("Oyo")

When to Use

  • When you need an ordered collection of items
  • When you need to modify your collection (add/remove items)
  • When items may have duplicates

Dictionaries

Key-value pairs for data mapping

Dictionaries are like phone books - look up information using a key (name) to find a value (number).

population = {
    "Lagos": 14.8,
    "Kano": 4.3,
    "Abuja": 3.6,
    "Rivers": 3.3
}

When to Use

  • When you need to store data that shouldn't change
  • For fixed collections like coordinates, RGB values
  • When you need faster iteration than lists

Tuples

Immordered, immutable sequences

Tuples are like sealed documents - once created, their content cannot be changed.

geo_coords = (6.5244, 3.3792)  # Lagos coordinates

When to Use

  • When you need to store labeled data
  • For fast lookups by key instead of position
  • When data has unique identifiers

Sets

Unordered, unique elements

Sets are like mathematical sets - they contain unique elements and support operations like union and intersection.

unique_products = {"oil", "cocoa", "rubber", "cocoa"}

When to Use

  • When you need to ensure all elements are unique
  • For membership tests (checking if item exists)
  • For mathematical set operations

Real-World Analogy

Data structures are like different storage solutions in a Nigerian home: Lists are like shelves where items are stored in order and can be rearranged. Dictionaries are like labeled storage boxes - you find things by their label (key). Tuples are like sealed packages that can't be changed once created. Sets are like a collection of unique spices - duplicates are automatically removed.

Practical Application: Student Database

Combining data structures to solve real problems

In data science, you'll often combine different data structures. Here's an example of managing student records:

# List of student dictionaries
students = [
    {
        "id": "ST001",
        "name": "Chinwe Okonkwo",
        "courses": {"Math", "Physics", "Computer Science"},
        "grades": (85, 92, 78)
    },
    {
        "id": "ST002",
        "name": "Tunde Okafor",
        "courses": {"Economics", "Statistics", "Computer Science"},
        "grades": (76, 88, 91)
    }
]

# Add a new student
new_student = {
    "id": "ST003",
    "name": "Amina Mohammed",
    "courses": {"Biology", "Chemistry", "Physics"},
    "grades": (92, 85, 89)
}
students.append(new_student)

# Find all students taking Computer Science
cs_students = [student["name"] for student in students 
               if "Computer Science" in student["courses"]]

print("Computer Science Students:", cs_students)

# Calculate average grade for a student
def calculate_average(grades_tuple):
    return sum(grades_tuple) / len(grades_tuple)

# Print student report
print("\nStudent Report:")
for student in students:
    avg = calculate_average(student["grades"])
    print(f"{student['name']}: {avg:.1f}% average")
Lines 1-14:

We create a list called students that contains dictionaries. Each dictionary represents a student with:

  • An ID (string)
  • Name (string)
  • Courses (a set of unique course names)
  • Grades (a tuple of immutable grades)
Line 18: students.append(new_student)

We add a new student to our list using the append() method. This demonstrates how lists can grow dynamically.

Lines 21-22: List comprehension

This compact syntax filters students who have "Computer Science" in their courses set and extracts their names.

Lines 25-26: Function definition

We define a function to calculate average grades. This shows how to create reusable code blocks.

Lines 29-32: Looping through students

We iterate through each student dictionary, calculate their average grade, and print a formatted report.

Common Mistakes

  • Modifying tuples:

    Attempting to change elements in a tuple (immutable)

  • Using wrong brackets:

    Confusing [] (lists), {} (dicts/sets), and () (tuples)

  • Assuming order in sets/dicts:

    Sets and dicts are unordered (use OrderedDict if order matters)

  • Key errors:

    Accessing dictionary keys that don't exist without checking

Best Practices

  • Choose the right structure:

    Select based on your data needs (mutable? ordered? unique?)

  • Use list comprehensions:

    For concise and readable list transformations

  • Leverage dictionary methods:

    .get() avoids KeyErrors, .items() for iteration

  • Use sets for uniqueness:

    Quickly remove duplicates from lists

File Handling

Reading and writing data

file_handling.py 
# Reading from a file (Nigeria census data)
with open('census_data.csv', 'r') as file:
    header = file.readline().strip().split(',')
    print(f"Columns: {header}")
    
    data = []
    for line in file:
        row = line.strip().split(',')
        data.append(row)

# Writing to a new file
with open('summary.txt', 'w') as output:
    output.write("Nigeria State Population Summary\n\n")
    for state_data in data[:5]:  # First 5 states
        output.write(f"{state_data[0]}: {state_data[1]} people\n")

📂 Python Census Code Explanation

Reading the file:

  • with open('census_data.csv', 'r'): Opens the CSV file containing census data in read mode.
  • file.readline().strip().split(','): Reads the first line (header), removes spaces, and splits it into column names.
  • for line in file:: Iterates through each subsequent line (each state’s data).
  • data.append(row): Appends each state’s data as a list into a new list called data.

Writing to a file:

  • with open('summary.txt', 'w'): Opens/creates summary.txt in write mode.
  • output.write(): Writes a heading and summary information to the new file.
  • for state_data in data[:5]: Loops over the first five rows in the data list (i.e., first 5 states).

Error Handling

Debugging and exceptions

error_handling.py 
# Safe division with error handling
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Error: Cannot divide by zero!")
        result = None
    except TypeError:
        print("Error: Please provide numbers")
        result = None
    else:
        print(f"Division successful: {a} / {b} = {result}")
    finally:
        print("Operation attempted\n")
    return result

# Test cases
safe_divide(100, 25)   # Valid division
safe_divide(50, 0)    # Divide by zero
safe_divide("100", 5)  # Type error

Explanation:

  • try: Attempts to divide a by b.
  • except ZeroDivisionError: Catches if b is 0 and avoids crashing.
  • except TypeError: Catches if the inputs aren't numbers.
  • else: Executes if no errors occur and prints the result.
  • finally: Always prints that the operation was attempted.
  • The function returns the result or None if there was an error.

Test cases:

  • safe_divide(100, 25) → Outputs result normally.
  • safe_divide(50, 0) → Catches divide-by-zero error.
  • safe_divide("100", 5) → Catches type mismatch.
