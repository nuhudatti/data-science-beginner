Introduction to Iris Dataset

The Iris flower data set is a multivariate data set introduced by the British statistician and biologist Ronald Fisher in his 1936 paper. It is sometimes called Anderson's Iris data set because Edgar Anderson collected the data to quantify the morphologic variation of Iris flowers of three related species.

Dataset Characteristics

  • Number of Instances: 150 (50 in each of three classes)
  • Number of Attributes: 4 numeric, predictive attributes and the class
  • Attribute Information:
    1. Sepal length in cm
    2. Sepal width in cm
    3. Petal length in cm
    4. Petal width in cm
    5. Class (Iris Setosa, Iris Versicolour, Iris Virginica)

Why Learn with the Iris Dataset?

The Iris dataset is perfect for beginners because:

Simple Structure

Only 4 features and 3 classes make it easy to understand

Visualization Friendly

Perfect for learning basic plotting techniques

Classification Benchmark

Commonly used to test machine learning algorithms

Multi-language Support

Available in both Python and R for comparison

Python Implementation

Python

1 Data Loading and Preparation

iris_analysis.py 
# Import necessary libraries
from sklearn.datasets import load_iris
import pandas as pd

# Load the Iris dataset from sklearn
iris = load_iris()

# Convert to pandas DataFrame
iris_df = pd.DataFrame(data=iris.data, columns=iris.feature_names)

# Add species names as a new column
iris_df['species'] = iris.target_names[iris.target]

Code Explanation:

  1. Import statements: We import the Iris dataset from scikit-learn and pandas for data manipulation.
  2. Loading data: load_iris() loads the dataset which contains numerical data and metadata.
  3. DataFrame conversion: We create a pandas DataFrame from the numerical data using the feature names as column headers.
  4. Adding species: The target values (0,1,2) are mapped to species names ('setosa', 'versicolor', 'virginica') and added as a new column.

Note: scikit-learn's Iris dataset is a Bunch object (similar to a dictionary) containing both data and metadata. We're extracting the numerical data and feature names to create a structured DataFrame.

Data Exploration

iris_analysis.py 
# Show first 10 rows
print("First 10 rows:")
print(iris_df.head(10))

# Dataset information
print("\nDataset information:")

# Statistical summary
print("\nStatistical summary:")
print(iris_df.describe())

# Check for missing values
print("\nMissing values per column:")
print(iris_df.isnull().sum())

# Count species occurrences
print("\nSpecies count:")
print(iris_df['species'].value_counts())

Code Explanation:

  1. head(10): Displays the first 10 rows of the DataFrame to inspect the data structure.
  2. info(): Provides metadata about the DataFrame including column data types and non-null counts.
  3. describe(): Generates descriptive statistics (count, mean, std, min, quartiles, max) for numerical columns.
  4. isnull().sum(): Checks for missing values in each column (important for data quality assessment).
  5. value_counts(): Counts occurrences of each species (categorical data analysis).

Pro Tip: These exploration methods form the foundation of the EDA (Exploratory Data Analysis) process. Always start with these basic checks to understand your dataset's structure and quality.

2 Data Analysis

iris_analysis.py 
# Average petal length per species
avg_petal_length = iris_df.groupby('species')['petal length (cm)'].mean()
print("\nAverage petal length per species:")
print(avg_petal_length)

# Species with longest average sepal length
longest_sepal_species = iris_df.groupby('species')['sepal length (cm)'].mean().idxmax()
print(f"\nSpecies with longest average sepal length: {longest_sepal_species}")

# Filter rows where petal width > 1.5
filtered_df = iris_df[iris_df['petal width (cm)'] > 1.5]
print("\nRows where petal width > 1.5:")
print(filtered_df.head())

Code Explanation:

  1. Groupby operation: groupby('species') groups data by species, then we calculate mean petal length for each group.
  2. idxmax(): After calculating mean sepal length by species, idxmax() returns the species name with the highest value.
  3. Filtering: The condition iris_df['petal width (cm)'] > 1.5 creates a boolean mask, used to filter rows where petal width exceeds 1.5cm.
Groupby Concept

Groupby splits data into groups based on a criterion, applies a function to each group independently, then combines results.

Boolean Masking

A powerful pandas feature where we filter data using True/False conditions applied to each row.

3 Data Visualization

Scatter Plot: Sepal Length vs Sepal Width

iris_analysis.py 
import matplotlib.pyplot as plt
import seaborn as sns

plt.figure(figsize=(8, 6))
sns.scatterplot(data=iris_df, x='sepal length (cm)', y='sepal width (cm)', hue='species', palette='viridis', s=100)
plt.title('Sepal Length vs Sepal Width by Species')
plt.xlabel('Sepal Length (cm)')
plt.ylabel('Sepal Width (cm)')
plt.legend(title='Species')
plt.grid(True, alpha=0.3)
plt.show()

Code Explanation:

  1. Imports: We import matplotlib for basic plotting and seaborn for enhanced visualization.
  2. Figure size: figsize sets dimensions (width, height in inches).
  3. Scatterplot: sns.scatterplot creates the plot with x/y axes, colors points by species (hue), uses viridis color palette, and sets point size (s).
  4. Labels/title: Self-explanatory labeling functions.
  5. Legend: Automatically created from hue mapping, with custom title.
  6. Grid: Adds light grid lines for better readability.

Histogram: Petal Length Distribution

plt.figure(figsize=(8, 6))
sns.histplot(data=iris_df, x='petal length (cm)', hue='species', element='step', kde=True, palette='viridis')

plt.xlabel('Petal Length (cm)')
plt.ylabel('Frequency')
plt.legend(title='Species')
plt.grid(True, alpha=0.3)
plt.show()

Code Explanation:

  1. histplot: Creates histogram with species differentiation via hue.
  2. element='step': Draws step-style histogram (alternative to bars).
  3. kde=True: Adds Kernel Density Estimate plot (smoothed distribution line).
  4. palette: Consistent coloring with scatter plot for comparison.

Boxplot: Sepal Width by Species

iris_analysis.py 
plt.figure(figsize=(8, 6))
sns.boxplot(data=iris_df, x='species', y='sepal width (cm)', palette='viridis')
plt.title('Sepal Width Distribution by Species')
plt.xlabel('Species')
plt.ylabel('Sepal Width (cm)')
plt.grid(True, alpha=0.3)
plt.show()

Code Explanation:

  1. boxplot: Creates box-and-whisker plot showing distribution statistics.
  2. x and y: Categorical variable (species) on x-axis, numerical (sepal width) on y-axis.
  3. Box components: The box shows IQR (25th-75th percentile), line at median, whiskers show range (excluding outliers).
  4. Outliers: Points beyond whiskers are plotted individually.

Boxplot Insight: This visualization helps compare distributions across categories, showing central tendency, dispersion, and outliers at a glance.

Basic Data Exploration

iris_analysis.R 
# Statistical summary
summary(iris)

# Check for missing values
colSums(is.na(iris))

# Count species occurrences
table(iris$Species)

Code Explanation:

  1. summary(): Provides descriptive statistics for each column (min, max, mean, median, quartiles for numeric; counts for factors).
  2. is.na() with colSums(): Checks for NA values (R's equivalent of null/NaN) in each column.
  3. table(): Creates a frequency table for categorical variables (Species in this case).

2 Data Analysis

iris_analysis.R 
# Average petal length per species
avg_petal_length <- aggregate(Petal.Length ~ Species, data = iris, FUN = mean)
print("Average petal length per species:")
print(avg_petal_length)

# Species with longest average sepal length
avg_sepal_length <- aggregate(Sepal.Length ~ Species, data = iris, FUN = mean)
longest_sepal_species <- avg_sepal_length$Species[which.max(avg_sepal_length$Sepal.Length)]
print(paste("Species with longest average sepal length:", longest_sepal_species))

# Filter rows where Petal.Width > 1.5
filtered_iris <- iris[iris$Petal.Width > 1.5, ]
print("Rows where Petal.Width > 1.5:")
print(head(filtered_iris))

Code Explanation:

  1. aggregate(): Splits data into subsets by Species and computes mean Petal.Length for each.
  2. which.max(): Finds the index of the maximum value in the average sepal length vector.
  3. Filtering: Uses logical indexing to select rows where Petal.Width exceeds 1.5.

R vs Python: Notice how R uses $ for column access and [ , ] for row/column selection, while Python uses [] and . notation. The aggregate function in R is similar to groupby in pandas.

3 Data Visualization

Scatter Plot: Sepal.Length vs Sepal.Width

iris_analysis.R 
# Base R graphics
plot(iris$Sepal.Length, iris$Sepal.Width, 
     col = iris$Species, pch = 19,
     main = "Sepal Length vs Sepal Width by Species",
     xlab = "Sepal Length (cm)",
     ylab = "Sepal Width (cm)")
legend("topright", legend = levels(iris$Species), 
       col = 1:3, pch = 19, title = "Species")

# Using ggplot2 (recommended)
library(ggplot2)
ggplot(iris, aes(x = Sepal.Length, y = Sepal.Width, color = Species)) +
  geom_point(size = 3) +
  labs(title = "Sepal Length vs Sepal Width by Species",
       x = "Sepal Length (cm)",
       y = "Sepal Width (cm)") +
  theme_minimal()

Code Explanation:

  1. Base R plot:
    • plot(x, y) creates scatter plot
    • col colors points by Species factor levels
    • pch=19 uses solid circles
    • legend() adds color key
  2. ggplot2 version:
    • ggplot() initializes plot with data and aesthetics
    • geom_point() adds scatter points
    • labs() sets labels
    • theme_minimal() applies clean theme

ggplot2 Advantage: While base R graphics are quick, ggplot2 provides more elegant, customizable, and consistent visualizations with a layered grammar of graphics approach.

Code Explanation:

  2. ggplot2 version:

    • position="identity" overlays histograms

Boxplot: Sepal.Width by Species

iris_analysis.R 
# Base R boxplot
boxplot(Sepal.Width ~ Species, data = iris,

Code Explanation:

  1. Base R boxplot:
    • boxplot(y ~ x) formula interface groups y by x
    • col sets box colors manually
  2. ggplot2 version:
    • geom_boxplot() creates the boxplot
    • alpha makes boxes slightly transparent
    • scale_fill_viridis_d() uses viridis color palette

Boxplot Interpretation: The box represents the interquartile range (IQR), the line inside is the median, whiskers show 1.5*IQR, and points are outliers.

Python vs R Comparison

After completing the analysis in both Python and R, let's compare the two approaches and highlight key differences:

Key Observations

Python Advantages

  • Pandas DataFrames offer very intuitive data manipulation syntax
  • Method chaining (df.method1().method2()) creates readable workflows
  • Seaborn provides high-level statistical visualization with minimal code
  • Better for production and integration with web applications
  • More consistent syntax across data operations

R Advantages

  • Built-in datasets like iris are immediately available
  • ggplot2 offers unparalleled customization and layered grammar
  • Statistical functions are more comprehensive out-of-the-box
  • Formula interface (y ~ x) is elegant for modeling
  • RMarkdown excels at reproducible research reports

When to Use Each

Scenario Preferred Language Reason
Production data pipelines Python Better integration with web frameworks and APIs
Academic research R More statistical methods readily available
Machine learning Python Stronger ecosystem (scikit-learn, TensorFlow, PyTorch)
Data visualization R ggplot2 offers more customization and publication-quality outputs
General-purpose programming Python More versatile beyond data analysis

Recommendation for Beginners

If you're just starting in data science, Python may be the better first language due to its general-purpose nature and growing dominance in industry. However, learning both will make you more versatile. Many professionals use R for exploratory analysis and visualization, then Python for production implementation.
