2. Calculates the arithmetic mean: sum of all values √∑ number of values
3. Returns a single numerical value representing the average

Example: If values are [5.2, 7.8, 3.5, 9.1, 4.3], mean = (5.2+7.8+3.5+9.1+4.3)/5 = 6.0

Potential Questions & Answers

Q: Why use df.head() instead of just printing the whole DataFrame?

A: For large datasets, printing the entire DataFrame would be overwhelming and slow. df.head() gives a quick preview without overwhelming the output.

Q: What's the difference between df.info() and df.describe()?

A: df.info() provides metadata about the DataFrame (column names, data types, non-null counts), while df.describe() provides statistical summaries of numerical columns (mean, std, min, max, etc.).

Q: Why use [0] when getting the mode?

A: The mode() method returns a Series of mode values. If there are multiple values with the same frequency (a multimodal distribution), [0] selects the first one. This ensures we always get a single value even when there are ties.

Q: When would value_counts() be more useful than mode()?

A: value_counts() shows the distribution of all values, not just the most frequent one. This is useful when you want to see the frequency of all categories, not just the most common one.

Q: Why format numbers to 2 decimal places?

A: Formatting to 2 decimal places makes the output more readable and professionally presented. For screen time and sleep duration, 2 decimal places provides appropriate precision without being overly specific.

Q: What if there are missing values in the data?

A: The .mean() method automatically excludes NaN (missing) values when calculating the average. However, .mode() and .value_counts() behave differently with missing data, so it's important to handle missing values before these operations.

Q: How could this analysis be extended?

A: This could be extended with visualizations (histograms, box plots), correlation analysis, grouping by categories, or more sophisticated statistical tests.
