Key Concepts

Data Visualization: The graphical representation of data to reveal patterns, trends, and relationships.

Exploratory Data Analysis (EDA): Using visual methods to explore and understand data before formal modeling.

Chart Types: Different visualizations (countplot, scatterplot, heatmap, stripplot) serve different purposes in data exploration.

Correlation Analysis: Examining how variables relate to each other, with values ranging from -1 (perfect negative correlation) to +1 (perfect positive correlation).

Important Notes

• Always call plt.figure() before creating a new plot to ensure a clean canvas

• Use appropriate figure sizes for different visualization types

• Color palettes should be chosen carefully for clarity and colorblind accessibility

• Heatmaps are particularly useful for identifying relationships between multiple variables

• Jitter in stripplots helps prevent overplotting when many points have similar values

• tight_layout() automatically adjusts subplot parameters to fit the figure area

Potential Questions & Answers

Q: Why use a countplot instead of a bar plot for categorical data?

A: Countplot is a specialized version of bar plot that automatically counts and plots the frequency of categorical variables, saving the step of manually calculating value counts before plotting.

Q: When would you use a scatterplot vs a stripplot?

A: Scatterplots are ideal for showing relationships between two continuous variables. Stripplots are better for showing the distribution of a continuous variable across different categories.

Q: What does the hue parameter do in Seaborn plots?

A: The hue parameter adds a semantic grouping by color based on another variable. This allows you to visualize relationships across three variables in a 2D plot.

Q: Why is jitter important in stripplots?

A: Jitter adds a small amount of random noise to the position of points, which helps prevent overlapping when many points have similar values. This makes the distribution easier to see.

Q: What does the correlation value tell us?

A: Correlation measures the strength and direction of a linear relationship between two variables. Values close to 1 or -1 indicate strong relationships, while values near 0 indicate weak or no linear relationship.

Q: Why use a heatmap for correlation instead of a table?

A: Heatmaps use color to represent values, making patterns and relationships easier to spot quickly. Our brains process visual information much faster than numerical data in tables.

Q: What's the purpose of the numeric_only=True parameter in df.corr()?

A: This parameter ensures that only numerical columns are included in the correlation calculation. Without it, pandas might try to compute correlations for non-numerical data, which could cause errors.

Q: How would you interpret a correlation heatmap?

A: Look for strong colors (red for positive, blue for negative) to identify strong relationships. The diagonal will always be 1 (each variable perfectly correlates with itself). The matrix is symmetric across the diagonal.
