Data Processing Pipeline

Step-by-step explanation of the data integration and cleaning process

1

Importing Essential Libraries

Explanation

This step imports the core Python libraries required for data manipulation and analysis:

  • pandas: Provides DataFrame structures for efficient data manipulation
  • numpy: Enables numerical operations and handling of missing values
  • These libraries form the foundation for handling diverse data formats

Research Significance: Ensures consistency in data handling methods across the team and provides standardized functions for data processing.

Implementation

import pandas as pd
import numpy as np

# pandas: Data manipulation and analysis
# numpy: Numerical computing and array operations

Best Practices:

  • Use standard aliases (pd, np) for consistency
  • Import only necessary modules to reduce memory overhead
  • Document library versions in requirements.txt
2

Loading Research Datasets

Explanation

This step loads three distinct research datasets from different sources:

  • CSV: Screen time data (nigerian_screen_time.csv)
  • Excel: Mental wellbeing survey results (mental_wellbeing_survey.xlsx)
  • JSON: Participant demographics (demographics.json)

Each dataset contains unique but complementary information about participants.

Research Significance: Maintains data integrity by loading each dataset with appropriate parsers. Different formats represent how data is collected from various sources in real research scenarios.

Implementation

# Load screen time data from CSV
screen_time = pd.read_csv('nigerian_screen_time.csv')

# Load wellbeing survey from Excel
wellbeing = pd.read_excel('mental_wellbeing_survey.xlsx')

# Load demographics from JSON
demographics = pd.read_json('demographics.json', orient='records')

# Display dataset summaries
print("Screen Time Data:", screen_time.shape)
print("Wellbeing Data:", wellbeing.shape)
print("Demographics Data:", demographics.shape)

Data Validation:

  • Check row counts match expected participant numbers
  • Verify critical columns exist in each dataset
  • Inspect data types for each column
3

Merging Research Datasets

Explanation

This step integrates the three datasets into a unified research dataset:

  • Datasets are merged using participant_id as the unique identifier
  • Screen time and wellbeing data are merged first
  • Demographic data is then added to the merged dataset
  • Inner join ensures only participants present in all datasets are included

Research Significance: Creates a comprehensive dataset where each row represents a complete participant record. This is essential for multivariate analysis examining relationships between screen time, wellbeing, and demographics.

Implementation

# Merge screen time and wellbeing data
merged_data = pd.merge(
  screen_time,
  wellbeing,
  on='participant_id',
  how='inner'
)

# Add demographic information
final_data = pd.merge(
  merged_data,
  demographics,
  on='participant_id',
  how='inner'
)

# Verify merged dataset
print("Final Dataset Shape:", final_data.shape)
print("Columns:", list(final_data.columns))

Merging Strategy:

  • Inner join preserves only complete cases across datasets
  • Alternative: Left join preserves all screen time participants
  • Check for duplicate participant_ids before merging
4

Handling Missing Values

Explanation

This step addresses missing data to ensure analysis validity:

  • Social Media Time: Missing values replaced with median
  • Wellbeing Score: Missing values replaced with mean
  • Median used for skewed distributions, mean for normal distributions
  • Preserves participant data while minimizing bias

Research Significance: Prevents loss of valuable participant data while minimizing potential bias from missing values. The choice of imputation method (median vs. mean) depends on variable distribution.

Implementation

# Calculate median social media time
social_median = final_data['social_media_time'].median()

# Impute missing social media values
final_data['social_media_time'] = final_data['social_media_time'].fillna(social_median)

# Calculate mean wellbeing score
wellbeing_mean = final_data['wellbeing_score'].mean()

# Impute missing wellbeing scores
final_data['wellbeing_score'] = final_data['wellbeing_score'].fillna(wellbeing_mean)

# Verify no missing values in key columns
print("Missing Values:")
print(final_data[['social_media_time', 'wellbeing_score']].isna().sum())

Advanced Options:

  • Regression imputation for multivariate relationships
  • Multiple imputation for more robust results
  • Create "missing" indicator variables

Frequently Asked Questions

Common questions about the data pipeline and research

Why use median for social media time but mean for wellbeing scores?

This approach accounts for the different distribution characteristics of these variables:

  • Social Media Time: Typically has a right-skewed distribution (many low users, few high users). The median is more robust to outliers in skewed data.
  • Wellbeing Scores: Usually follow a normal distribution where the mean accurately represents the central tendency.
  • Using distribution-appropriate measures prevents bias in our imputation.

How were participants recruited for this study?

Participants were recruited through:

  • University participant pools
  • Community outreach programs in Lagos and Abuja
  • Online recruitment targeting diverse age groups
  • Stratified sampling to ensure demographic representation

Inclusion criteria: 18+ years, smartphone ownership, willingness to install tracking app.

What ethical considerations were addressed in this research?

This study followed strict ethical guidelines:

  • Institutional Review Board (IRB) approval obtained
  • Informed consent from all participants
  • Data anonymization through participant IDs
  • Secure data storage with encryption
  • Right to withdraw at any time without penalty
  • Mental health resources provided for distressed participants

How can I extend this analysis with machine learning?

The clean dataset provides an excellent foundation for ML approaches:

  • Classification: Predict wellbeing categories (low/medium/high) from screen patterns
  • Regression: Predict continuous wellbeing scores
  • Clustering: Identify participant segments with similar screen time behaviors
  • Feature Engineering: Create new features like "social media ratio" or "evening usage"

We provide example Jupyter notebooks for these extensions in the research repository.
